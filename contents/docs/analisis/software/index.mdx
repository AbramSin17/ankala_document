## 1. Pendahuluan

## 1.1 Tujuan 
Dokumen ini bertujuan untuk mendefinisikan kebutuhan fungsional dan non-fungsional dari website ankalajourney.site yang menyediakan jasa tour guide untuk wisatawan.

1.2 Lingkup 
Website ini akan memungkinkan pengguna untuk:
- Melihat daftar jasa tour guide yang tersedia.
- Melakukan pemesanan jasa tour guide.
- Mengelola profil pengguna.
- Memberikan ulasan dan penilaian terhadap jasa yang diterima.

1.3 Definisi, Akronim, dan Singkatan
- Tour Guide: Penyedia jasa yang memandu wisatawan.
- User: Pengguna yang mengakses website untuk mencari dan memesan jasa tour guide.
- Admin: Pengelola website yang mengelola konten dan pengguna.

## 2. Deskripsi Umum

## 2.1 Perspektif Produk 
Website ini akan menjadi platform yang menghubungkan wisatawan dengan tour guide. Website ini akan memiliki antarmuka yang ramah pengguna dan responsif.

## 2.2 Fitur Utama
- Pendaftaran dan login pengguna.
- Pencarian dan filter tour guide berdasarkan lokasi, jenis tur, dan harga.
- Halaman detail untuk setiap tour guide.
- Sistem ulasan dan penilaian.
- Panel admin untuk mengelola konten dan pengguna.
- Penambahan dan penghapusan favorite tourguide 
- Pengolahan akun pengguna

## 3. Kebutuhan Fungsional

## 3.1 Pengguna
- Pendaftaran dan Login
    - Pengguna dapat mendaftar dengan email dan password.
    - Pengguna dapat login menggunakan email dan password.
- Pencarian Tour Guide
    - Pengguna dapat mencari tour guide berdasarkan lokasi, jenis tur, dan harga.
    - Pengguna dapat melihat daftar tour guide yang sesuai dengan kriteria pencarian.
- Detail Tour Guide
    - Pengguna dapat melihat detail tour guide, termasuk deskripsi, harga, dan ulasan.
- Pemesanan
    - Pengguna dapat memesan jasa tour guide.
    - Pengguna dapat memilih tanggal dan waktu untuk pemesanan.
- Pembayaran
    - Pengguna dapat melakukan pembayaran melalui metode yang tersedia (kartu kredit, transfer bank, dll).
- Ulasan dan Penilaian
    - Pengguna dapat memberikan ulasan dan penilaian setelah menggunakan jasa tour guide.

## 3.2 Admin
- Manajemen Pengguna
    - Admin dapat melihat dan mengelola daftar pengguna.
- Manajemen Tour Guide
    - Admin dapat menambahkan, mengedit, dan menghapus informasi tour guide.
- Manajemen Ulasan
    - Admin dapat melihat dan mengelola ulasan yang diberikan oleh pengguna.

## 4. Kebutuhan Non-Fungsional

## 4.1 Kinerja
Website harus dapat menangani minimal 100 pengguna secara bersamaan.

## 4.2 Keamanan
Data pengguna harus dienkripsi dan dilindungi.
Sistem harus memiliki mekanisme untuk mencegah akses tidak sah.

## 4.3 Usability
Antarmuka pengguna harus intuitif dan mudah digunakan.
Website harus responsif dan dapat diakses di berbagai perangkat (desktop, tablet, smartphone).

## 4.4 Kompatibilitas
Website harus kompatibel dengan browser utama (Chrome, Firefox, Safari, Edge).

## 5. Batasan
Website ini hanya akan beroperasi di wilayah tertentu (misalnya, satu negara atau kota).
Pembayaran hanya dapat dilakukan dalam mata uang lokal.
